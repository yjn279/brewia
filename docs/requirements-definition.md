# Brewia 要件定義書（現行実装ベース）

最終更新: 2026-03-28  
対象: 現在の `app/` / `components/` 実装で確認できる要件

---

## 1. 文書目的

本書は、Brewia の現行実装から読み取れる**要件（何を満たすべきか）**を定義する。  
画面構成・項目詳細・遷移の具体仕様は `docs/screen-specification.md` を参照。

---

## 2. システム概要

Brewia は、コーヒー豆（Bean）と抽出（Brew）の体験を記録し、豆単位・抽出単位で履歴参照できるモバイル指向アプリである。

---

## 3. スコープ

### 3.1 対象機能（In Scope）

1. 豆情報の登録
2. 抽出ログの登録
3. ホームでの統計サマリー表示
4. 豆詳細の閲覧（豆情報 + 紐づく抽出履歴）
5. 抽出詳細の閲覧（抽出条件 + 味覚評価 + フレーバー + ノート）
6. URL クエリによる新規作成画面の初期状態制御

### 3.2 非対象（Out of Scope / 未実装）

1. 永続化DBへの保存
2. `/beans` 一覧画面
3. `/brews` 一覧画面
4. 認証・認可
5. 外部連携

---

## 4. 機能要件

### FR-01 豆登録
- ユーザーは豆の基本情報・産地情報・特性・メモを入力し登録できること。
- 必須項目は Name / Roaster / Country とすること。

### FR-02 抽出登録
- ユーザーは対象豆を選択し、抽出パラメータ・味覚評価・フレーバー・メモを入力し登録できること。
- 必須項目は Bean / Coffee / Water / Temp / Grind とすること。

### FR-03 サマリー表示
- ホーム画面で以下を表示できること。
  - Total Brews
  - Bean Varieties
  - Countries
  - Avg Rating

### FR-04 豆詳細表示
- 豆詳細画面で、豆の概要・産地情報・メモ・抽出履歴を表示できること。

### FR-05 抽出詳細表示
- 抽出詳細画面で、抽出条件・注湯プロファイル・味覚レーダー・フレーバー・ノートを表示できること。

### FR-06 画面遷移
- ホームから新規作成、豆詳細へ遷移できること。
- 豆詳細から抽出新規作成、抽出詳細へ遷移できること。
- 戻る導線を各詳細/新規画面に持つこと。

### FR-07 クエリパラメータ制御
- `/new?type=bean|brew` で初期タブを制御できること。
- `/new?type=brew&bean=<id>` で初期選択豆を指定できること。

### FR-08 不正ID時のエラー処理
- `/beans/[id]` または `/brews/[id]` で対象データが存在しない場合は Not Found を返すこと。

---

## 5. 非機能要件

### NFR-01 UI/UX
- モバイルファースト（単一カラム中心）であること。
- タッチしやすいボタンサイズと視認性を確保すること。
- カード分割による情報の意味的グルーピングを行うこと。

### NFR-02 入力品質
- 必須項目や数値範囲はフォーム側で基本バリデーションを提供すること。
- 数値は単位（g, °C, clicks）を明示すること。

### NFR-03 拡張性
- 型中心でデータ構造を定義し、将来的な永続化差し替えに耐えられること。

### NFR-04 可用性
- 主要画面は直接URLアクセス可能であること。
- 不正パラメータ時に破綻せず、適切に Not Found または安全な既定動作を行うこと。

---

## 6. データ要件（論理）

### 6.1 Bean
- 識別: `id`
- 基本: `name`, `roaster`, `roast`, `notes`
- 産地: `country`, `region`, `farm`
- 特性: `variety`, `process`
- 管理: `updated`

### 6.2 Brew
- 識別: `id`
- 参照: `bean`
- 抽出条件: `beanWeight`, `waterWeight`, `waterTemp`, `beanGrind`, `steps`
- 評価: `aroma`, `acidity`, `sweetness`, `body`, `overall`
- 補足: `flavors`, `notes`, `created`
