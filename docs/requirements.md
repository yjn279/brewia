# 要件定義書

Brewia は、コーヒー豆の出会いと抽出体験を蓄積するパーソナル記録アプリである。本書は、これまでに完了した GitHub Issue・PR・コミット履歴を仕様の情報源とし、現行プロダクトが満たす要件を再構築したものである。

## プロダクト概要

### ビジョン

世界各国の豆との出会いと、その豆をどう抽出してどう味わったかを、単なる記録ではなく「コーヒー体験の航海日誌」として蓄積する。

### 主要ユースケース

アプリが対応する主なユースケースは次のとおり。

- 豆を登録する（手入力 / 写真から自動抽出）
- 豆ごとの抽出レシピと結果カップを記録する
- 抽出時に時間をラップ計測し、注湯ステップとして残す
- レシピをプリセット化し、次回の抽出登録で再利用する
- 抽出に味覚評価とフレーバータグを付与する
- 過去の抽出履歴を豆ごとに振り返る

### 想定ユーザー

- 自宅や趣味でハンドドリップ抽出を行うコーヒー愛好家
- スペシャルティコーヒー豆を継続的に試したいユーザー
- 抽出条件と味の関係を可視化したいユーザー

## 機能要件

### ユーザー認証 / アクセス制御

- Google OAuth によるサインイン / サインアウト
- セッションは Turso（libSQL/SQLite）に永続化
- `/login` と `/api/auth/*` 以外のページ・API はログイン必須。未認証は `/login` にリダイレクト
- すべての豆 / 抽出 / プリセットはユーザー単位で完全に分離
- 他ユーザーのリソースには 404 を返し、リソースの存在を伏せる

### 豆の管理

| サブ機能 |
| --- |
| 豆の新規登録 / 編集 / 削除 |
| 国・地域・農園・品種・精製・焙煎度・ロースター・メモ・価格 (JPY) の保持 |
| 国は固定リストから選択（19カ国 + Blended） |
| 焙煎度は 8 段階（Light / Cinnamon / Medium / High / City / Full City / French / Italian） |
| 焙煎度入力時にカラーパレットを表示 |
| 豆の写真から各フィールドを LLM で自動抽出 |
| 豆の写真から焙煎度を CIELAB L\* 曲線で自動推定 |
| 豆の削除時に紐づく抽出 / フレーバー紐付けも整合的に削除 |
| 既存の豆をコピーして新規登録のひな形にする |

### 抽出ログの管理

| サブ機能 |
| --- |
| 抽出の新規登録 / 編集 / 削除 |
| 入力項目: 豆量・湯量・湯温・挽き目・注湯ステップ・5 軸味覚評価・フレーバー・メモ |
| 注湯ステップは `(time, water)` の配列で保持し、面グラフで可視化 |
| 抽出登録時にカップ評価をスキップして「レシピのみ」状態で保存可能 |
| 既存の抽出をコピーして新規登録のひな形にする |
| 数値フィールドにインライン単位サフィックス（g, ml, °C 等） |
| 詳細画面で `null` 値が「null°C」等として表示されないようガード |

### 抽出タイマー

抽出登録フォーム上で動作するラップタイマーを搭載する。

- センチ秒（1/100 秒）まで表示
- ラップ時に直近の経過秒を 5 秒単位に丸めて注湯ステップ行へ反映
- Stop / Reset の 2 段階フロー
- Reset 押下時、ステップ行も初期化される旨の確認ダイアログを表示

### 抽出プリセット

抽出プリセットは、抽出レシピを名前付きで保存・再利用するための機能である。プリセットは以下の項目で構成される。

- 抽出レシピを名前付きプリセットとして保存 / 編集 / 削除
- プリセット項目: 名前・説明・既定湯温・抽出比率（豆量と湯量から算出）・抽出ステップ配列
- プリセットは `/presets` で一覧表示 / 個別編集
- プリセット保存ダイアログは抽出フォームの `Extraction Steps` ヘッダから起動
- ユーザー作成プリセットのみを表示
- プリセット編集ダイアログの抽出ステップ入力 UI は新規抽出フォームと同形

### フレーバー

- フレーバーは全ユーザー共通のマスタとして共有
- 抽出 1 件に複数のフレーバーをタグ付け可能（中間テーブル `brew_flavor`）
- フレーバーマスタは `GET /api/flavors` で取得（読み取り専用）

### ホーム / 統計

- ログイン後のホームに「Your Coffee Journey」サマリ（Total Brews / Bean Variety）を表示
- 豆の一覧をカード形式で表示し、空状態のときは登録誘導 CTA を出す
- ヘッダから Presets / 豆登録 / ユーザーメニューへ遷移

### PWA

- マニフェスト `/manifest.webmanifest` と Service Worker `/sw.js` を配信
- ホーム画面追加でスタンドアロン表示
- 静的アセットは Cache First、その他 GET は Network First、`/api/*` は素通し
- ナビゲーション失敗時は `/offline` フォールバック

## 非機能要件

### セキュリティ / マルチテナント

- DB アクセスはユーザー単位で分離し、他ユーザーのデータは取得できない構造にする
- 未認証アクセスには 401 を返し、未認証ページアクセスは `/login` にリダイレクトする
- 他ユーザーのリソースには存在情報を漏らさないよう 404 を返す
- フレーバーマスタは全ユーザー共通として扱い、ユーザー単位の所有は持たない

### データ整合性

- 豆削除時は紐づく抽出とフレーバー紐付けをトランザクション内で連鎖削除する
- 抽出更新時はフレーバー紐付けを一括削除してから再作成する
- スキーマ列は NULL 値を持たず、空入力は空文字または 0 で表現する

### 入力バリデーション

- すべての作成 / 更新 API はサーバー側でバリデーションを通す
- 国 / 焙煎度は固定リストの値のみ受け付ける
- 味覚評価は 1–5、湯温は 1–100 の整数。0 は空文字として扱う
- 写真アップロードは JPEG / PNG のみ、サーバー側上限 4.5 MB

### LLM 連携

- Claude API を使用して豆写真からフィールドを自動抽出する
- 失敗時は API エラーとパースエラーを区別してユーザーに通知する
- 自動抽出エンドポイントも認証必須とし、コスト発生を防ぐ

### 動作環境

- Web ブラウザ（モバイル幅を基準にしたレイアウト）
- PWA としてホーム画面に追加可能

## 用語集

| 用語 | 定義 |
| --- | --- |
| Bean | 焙煎済みコーヒー豆の登録レコード |
| Brew | 1 回の抽出ログ。豆量・湯量・ステップ・評価を持つ |
| Brew Step | 注湯ステップ。経過秒 `time` と累計湯量 `water` の組 |
| Brew Preset | 抽出レシピのひな形。名前・既定値・ステップを保持 |
| Flavor | 風味タグ（共有マスタ） |
| BrewFlavor | Brew と Flavor の中間テーブル |
| User | Auth.js が管理する認証ユーザー |

## スコープ外

| 項目 | 理由 |
| --- | --- |
| ユーザー間データ共有 / 公開 | プライベート記録に限定 |
| チーム / 組織テナンシー | 単一ユーザー前提 |
| ロール（admin 等） | 認可は ownership のみ |
| パスワード認証 | OAuth に限定 |
| プッシュ通知 / バックグラウンド同期 | PWA スコープ外 |
| 動的 API レスポンスの永続キャッシュ | オフライン書き込み非対応 |
| プロフィール編集画面 | 最小表示のみ |
