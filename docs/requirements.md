Brewia は個人向けコーヒー抽出ログアプリです。スマートフォンブラウザで動作する Progressive Web App として設計されており、豆の管理・抽出レシピの記録・テイスティング評価を一元管理できます。

## Background

コーヒー愛好家が抽出レシピや豆の情報を手軽に記録・参照できるよう、PWA として開発されています。オフライン対応と写真からの自動情報抽出（LLM 活用）が特徴です。

## Target users

Brewia の想定ユーザーは、ハンドドリップ・エアロプレス・フレンチプレスなどの手淹れコーヒーを日常的に楽しむ個人ユーザーです。スマートフォンから手軽に記録したい人を主なターゲットとします。

## Use cases

豆を登録する場面では、ユーザーは生産国・農園・品種・精製方法・焙煎度・価格などの情報を入力します。写真をアップロードすると LLM が豆情報を自動抽出し、フォームに流し込みます。写真の色解析による焙煎度推定（Lab 色空間の L\* 値使用）も合わせて行います。

抽出ログを作成する場面では、使用した豆を選択し、コーヒー量・水量・水温・グラインド設定・各注水ステップ（時間と水量）を記録します。ステップはドラッグ＆ドロップで並び替えできます。固定プリセット（Hario V60 4:6 など 5 種）から初期値を挿入することもできます。

テイスティング評価では、アロマ・酸味・甘さ・ボディ・総合評価を 1〜5 のスライダーで記録し、風味フレーバーをタグ形式で選択します。あとで記録モードでは評価を後回しにできます。

写真からの抽出は、`/api/beans/extract` エンドポイントで Anthropic Claude API を利用して豆情報（名前・ロースター・生産国・農園・品種・精製方法・ノート）を抽出します。

認証は、メール＋パスワード方式のサインアップ・ログイン・ログアウトを提供します。セッションは HttpOnly Cookie で管理されます。

## Non-functional requirements

PWA 対応として、`next.config.mjs` でオフラインページ（`/offline`）を提供し、Service Worker により基本的なオフラインフォールバックが機能します。

型安全性の確保のため、`pnpm exec tsc --noEmit` が 0 エラーで終了することを要件とします。Lint は `eslint` で実施し、新規 error が発生しないことを要件とします。テストは `vitest` で実施し、すべてのテストがパスすることを要件とします。

データの整合性は、Drizzle ORM のスキーマ定義と Zod バリデーションにより保証します。認証が必要なすべての API ルートおよびページで、未認証時に 401 または `/login` リダイレクトを返します。

## Out of scope

本リリース時点で意図的に対象外とした機能を以下に示します。

- パスワードリセット・メール検証・OAuth/SSO・2FA
- ロール・権限・組織・チーム機能
- Remember me・セッション延長 UX
- ユーザー設定ページ（プロフィール変更等）
- Flavor など Bean/Brew 以外のテーブルの user スコープ化
- AI による抽出プリセット自動生成
- 多言語対応・テーマカラー切替の拡張
- PATCH API による部分更新
- マイグレーションのロールバックスクリプト
