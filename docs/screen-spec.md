Brewia の画面構成は Next.js App Router に基づいています。認証が必要なページは未認証時に `/login` へリダイレクトします。以下では各画面の目的・主要 UI 要素・遷移先・認証要件を説明します。

## /login

ログイン画面です。

メール入力欄とパスワード入力欄、Sign in ボタン、「Create account」リンクを提供します。認証不要でアクセス可能。ログイン成功後は `/` へリダイレクトします。

## /signup

サインアップ画面です。

メール入力欄とパスワード入力欄（8 文字以上）、「Create account」ボタン、「Sign in」リンクを提供します。認証不要でアクセス可能。アカウント作成成功後は `/` へリダイレクトします。

## /

ホーム画面です。認証必須。

ヘッダーにアプリ名「Brewia」、サインアウトボタン（LogOut アイコン）、新規豆追加ボタン（`+`）を表示します。本体には挨拶メッセージ（時刻に応じて変化）、統計カード（総抽出数・豆の種類数）、登録済み豆の一覧（BeanCard）を表示します。豆がない場合は空状態の案内を表示します。各 BeanCard クリックで `/beans/[id]` へ遷移します。

## /beans/[id]

豆詳細画面です。認証必須。自分が登録した豆のみ閲覧可能。

ヘッダーには戻るボタン（`/`）、コピーボタン、編集ボタン、削除ボタン、新規抽出ボタン（`+`）を配置します。本体には豆のヒーローセクション（国旗・名前・ロースター・価格・焙煎度）、オリジン情報（地域・生産国・農園・品種・精製方法）、ノートセクション、抽出履歴（BrewCard 一覧）を表示します。

## /beans/[id]/edit

豆編集画面です。認証必須。自分の豆のみ編集可能。

`NewBeanForm` を `mode="edit"` で表示します。保存後は `/beans/[id]` へ戻ります。

## /new

新規登録画面です。認証必須。

URL パラメーターに応じて豆登録フォームまたは抽出ログ登録フォームを表示します（`?type=bean` or `?type=brew`）。`?copyBean=` または `?copyBrew=` パラメーターで既存データをコピーして初期値を設定できます。

豆登録フォーム（`NewBeanForm`）は写真から入力ボタン・豆情報・オリジン・特性・焙煎度（カラーパレットと写真から推定）・ノートの各セクションを提供します。

抽出ログ登録フォーム（`NewBrewForm`）は豆選択・パラメーター（コーヒー量・水量・温度・グラインド）・抽出ステップ（D&D 並び替え対応、プリセット挿入ボタン付き）・カップ評価・フレーバー・ノートの各セクションを提供します。

## /brews/[id]

抽出ログ詳細画面です。認証必須。自分の抽出ログのみ閲覧可能。

ヘッダーには豆詳細への戻るボタン、コピーボタン、編集ボタン、削除ボタンを配置します。本体には豆参照リンク（フラグ・名前・ロースター・総合評価）、パラメーターグリッド（コーヒー量・水量・温度・グラインド・抽出比率）、注水プロファイルチャート（PourChart）、テイストプロファイル（TasteRadar、overall > 0 のときのみ）、フレーバーノート、テイスティングノートを表示します。

## /brews/[id]/edit

抽出ログ編集画面です。認証必須。自分の抽出ログのみ編集可能。

`NewBrewForm` を `mode="edit"` で表示します。保存後は `/brews/[id]` へ戻ります。

## /offline

オフラインフォールバック画面です。認証不要。

Service Worker がオフライン時にフォールバックとして表示します。「You are offline」等の案内メッセージを表示します。
