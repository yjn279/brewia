# Brewia 要件定義書

## 1. 目的

Brewia は、豆情報と抽出ログを一元管理し、再現性のある抽出改善を支援する。
本書は、開発・QA が同一基準で機能実装と受け入れ判定を行えるよう、機能要件と受け入れ条件を定義する。

## 2. 対象範囲

- 豆管理（一覧/詳細/作成/コピー/編集/削除）
- 抽出管理（一覧/詳細/作成/コピー/編集/削除）
- フレーバー管理（一覧/選択）
- ダッシュボード表示（総抽出数、総豆数）

## 3. 機能要件

### 3.1 豆管理機能

#### 3.1.1 管理項目

| 和名     | 英名    | 型          | 必須 | 仕様                          |
| -------- | ------- | ----------- | ---- | ----------------------------- |
| 名称     | Name    | string      | ○    | 前後空白を除去し 1 文字以上。 |
| 生産国   | Country | enum        | ○    | 許可された国一覧から選択。    |
| 生産地域 | Region  | string      | -    | 空文字可。                    |
| 生産農園 | Farm    | string      | -    | 空文字可。                    |
| 生産処理 | Process | enum/string | -    | 定義済み候補またはその他。    |
| 品種     | Variety | string      | -    | 空文字可。                    |
| 焙煎度   | Roast   | enum        | ○    | 8 段階から選択。              |
| 焙煎所   | Roaster | string      | ○    | 前後空白を除去し 1 文字以上。 |
| メモ     | Note    | string      | -    | 空文字可。                    |
| 作成日時 | Created | datetime    | 自動 | 作成時に自動設定。            |
| 編集日時 | Updated | datetime    | 自動 | 更新時に自動更新。            |

#### 3.1.2 表示要件

- 生産国は国旗を併記する。
- ブレンドの場合は `🏳️‍🌈` を表示する。
- 焙煎度は以下の 8 レベルと色コードで表示する。
  - Light Roast: `#BA8D5D`
  - Cinnamon Roast: `#AD7C4E`
  - Medium Roast: `#9E6C41`
  - High Roast: `#8D5D37`
  - City Roast: `#7B4F2F`
  - Full City Roast: `#68432B`
  - French Roast: `#533728`
  - Italian Roast: `#3D2D27`
- 生産処理の推奨候補は以下とする。
  - ナチュラル / ウォッシュト / パルプトナチュラル / レッドハニー / イエローハニー / ホワイトハニー / アナエロビック / その他

#### 3.1.3 受け入れ条件（テスト観点）

- 必須項目未入力で保存した場合、保存されずエラーとなること。
- 生産国が未定義値の場合、保存されずエラーとなること。
- コピー作成時、元データを初期値として新規 ID で作成されること。
- 削除時、確認後に対象が削除され、一覧から消えること。

### 3.2 抽出管理機能

#### 3.2.1 管理項目

| 和名         | 英名         | 型          | 必須 | 仕様                                      |
| ------------ | ------------ | ----------- | ---- | ----------------------------------------- |
| 豆量         | Bean Weight  | number      | ○    | 正の数（g）。                             |
| 挽き目       | Bean Grind   | number/null | -    | クリック数。未入力可。                    |
| 湯量         | Water Weight | number      | ○    | 正の数（g）。                             |
| 湯温         | Water Temp   | number/null | -    | 0〜100（℃）、未入力可。                   |
| 抽出ステップ | Brew Steps   | array       | ○    | `[{time, water}]`、time/water は 0 以上。 |
| 香り         | Aroma        | int         | ○    | 1〜5。                                    |
| 酸味         | Acidity      | int         | ○    | 1〜5。                                    |
| 甘味         | Sweetness    | int         | ○    | 1〜5。                                    |
| 質感         | Body         | int         | ○    | 1〜5。                                    |
| 総合点       | Overall      | int         | ○    | 1〜5。                                    |
| メモ         | Note         | string      | -    | 空文字可。                                |
| 作成日時     | Created      | datetime    | 自動 | 作成時に自動設定。                        |
| 編集日時     | Updated      | datetime    | 自動 | 更新時に自動更新。                        |

#### 3.2.2 表示要件

- 豆量と湯量から抽出比率（Brew Ratio = `waterWeight / beanWeight`）を表示する。
- 抽出ステップは折れ線グラフで描画する。
  - 横軸: 時間（s）
  - 縦軸: 湯量（g）
  - 各点にステップ番号を表示
  - 時間・湯量の補助線を表示
- 味覚評価（香り/酸味/甘味/質感/総合点）はレーダーチャートで表示する。

#### 3.2.3 受け入れ条件（テスト観点）

- 豆量または湯量が 0 以下の場合、保存されずエラーとなること。
- 評価値が 1〜5 の範囲外の場合、保存されずエラーとなること。
- 抽出詳細で抽出比率が正しく計算・表示されること（小数 1 桁）。
- 抽出削除時、確認後に対象が削除され、詳細画面へ再アクセスで 404 になること。

### 3.3 フレーバー管理機能

#### 3.3.1 管理項目

| 和名         | 英名        | 型       | 必須 | 仕様               |
| ------------ | ----------- | -------- | ---- | ------------------ |
| 名称         | Name        | string   | ○    | フレーバー名。     |
| カテゴリ     | Category    | string   | ○    | 大分類。           |
| サブカテゴリ | Subcategory | string   | ○    | 小分類。           |
| 作成日時     | Created     | datetime | 自動 | 作成時に自動設定。 |
| 編集日時     | Updated     | datetime | 自動 | 更新時に自動更新。 |

#### 3.3.2 受け入れ条件（テスト観点）

- 抽出作成/編集画面でフレーバー一覧が表示されること。
- 複数選択したフレーバーが保存され、詳細画面で表示されること。

### 3.4 ダッシュボード機能

#### 3.4.1 表示要件

- 総抽出数（Total Brews）を表示する。
- 総豆数（Total Beans）を表示する。
- 豆一覧を表示する。
- 豆未登録時は空状態表示と豆作成導線を表示する。

#### 3.4.2 受け入れ条件（テスト観点）

- 豆・抽出登録後に件数表示が即時反映されること。
- 豆 0 件時に空状態が表示されること。
